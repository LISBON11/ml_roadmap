## Блок 2.1 — Рабочий процесс и постановка задачи `[A]`

**Зачем:** без правильной схемы эксперимента любой результат — самообман; утечка «рисует» отличные метрики, которых в проде не будет.

**2.1.1 Типы задач** — классификация, регрессия, ранжирование, кластеризация; объекты, признаки, target. 🎯 *Докажи себе:* для трёх задач определи тип и признаки/таргет.

**2.1.2 Train / validation / test** — зачем три выборки, почему на тест нельзя «подглядывать», hold-out. 🎯 *Докажи себе:* раздели датасет в sklearn и объясни роль каждой части.

**2.1.3 Кросс-валидация** — k-fold, стратификация. 🎯 *Докажи себе:* прогони 5-fold CV и сравни с hold-out.

**2.1.4 Переобучение и недообучение** — bias–variance, кривые обучения. 🎯 *Докажи себе:* переобучи дерево и покажи разрыв train/test.

**2.1.5 Утечка данных** ⭐ — источники (масштабирование до сплита, утечка таргета, временна́я). 🎯 *Докажи себе:* пример, где `StandardScaler` до сплита создаёт утечку.

**2.1.6 sklearn API и Pipeline** ⏭️ — `fit/predict/transform`, `Pipeline`, `ColumnTransformer`. 🎯 *Докажи себе:* собери Pipeline «препроцессинг → модель» одним `.fit()`.

**Ресурсы:** ML Handbook (основные понятия, измерение качества) — RU; Kaggle Learn Intro to ML — EN.

**Контрольное:** собери Pipeline с `ColumnTransformer`, оцени через 5-fold CV, объясни в 2 строках, где могла бы быть утечка и почему её нет.

## Блок 2.2 — Метрики качества `[A]`

**Зачем:** «A лучше B» — всегда про конкретную метрику. Задел под eval в Этапе 4.

**2.2.1 Матрица ошибок и базовые метрики** — confusion matrix, accuracy, precision, recall, F1; почему accuracy врёт при дисбалансе. 🎯 *Докажи себе:* придумай датасет 99/1, где accuracy 0.99 — плохая модель.

**2.2.2 ROC-AUC и PR-кривая** — TPR/FPR, AUC, precision-recall, когда какая. 🎯 *Докажи себе:* построй ROC, объясни AUC = 0.5.

**2.2.3 Порог классификации** — как сдвиг меняет precision/recall. 🎯 *Докажи себе:* подбери порог под recall ≥ 0.9, посмотри на precision.

**2.2.4 Метрики регрессии** — MAE, MSE, RMSE, R²; чувствительность к выбросам. 🎯 *Докажи себе:* MAE vs RMSE при одном большом выбросе.

**2.2.5 Метрики ранжирования (обзор)** ⭐ — precision@k, recall@k, MAP, NDCG. 🎯 *Докажи себе:* почему для поиска/RAG важнее precision@k.

**Ресурсы:** ML Handbook (измерение качества) — RU; **StatQuest** (Josh Starmer, YouTube, EN) для визуальной интуиции ROC/precision-recall — *проверено на июнь 2026.*

**Контрольное:** обучи две модели на несбалансированных данных, выбери лучшую **не по accuracy**, обоснуй метрику бизнес-смыслом.

## Блок 2.3 — Линейные модели `[A+B]`

**Зачем:** «hello world» ML и фундамент: логистическая регрессия = один нейрон с сигмоидой, мост к Этапу 3.

**2.3.1 Линейная регрессия** `[A]` — модель, MSE, аналитическое решение vs градиентный спуск. 🎯 *Докажи себе:* обучи `LinearRegression`, сверь с реализацией спуска из веха-проекта Этапа 1.

**2.3.2 Логистическая регрессия** `[A]` ⭐ — сигмоида, вероятность из линейного выхода, log-loss. 🎯 *Докажи себе:* почему выход всегда в (0,1).

**2.3.3 Регуляризация** `[A]` — L2 (Ridge), L1 (Lasso), ElasticNet; разреженность L1. 🎯 *Докажи себе:* увеличивай L1 и покажи обнуление коэффициентов.

**2.3.4 Масштабирование признаков** `[A]` — почему нужен один масштаб. 🎯 *Докажи себе:* сравни коэффициенты до/после `StandardScaler`.

**2.3.5 Мультиклассовость** `[B]` — one-vs-rest, softmax-регрессия. 🎯 *Докажи себе:* как бинарная логрегрессия обобщается на 3 класса.

**2.3.6 Вывод и связь с MLE** `[B]` — почему log-loss, вывод градиента, максимальное правдоподобие. 🎯 *Докажи себе:* выпиши градиент log-loss по весам.

**Ресурсы:** ML Handbook (линейные модели) — RU; mlcourse.ai — RU; **лекции К.В. Воронцова** (machinelearning.ru, видео ШАД), *RU, материалы обновляются 2025–2026* — http://www.machinelearning.ru/wiki/index.php?title=Машинное_обучение_(курс_лекций,_К.В.Воронцов) — *проверено на июнь 2026.*

**Контрольное:** сравни логрегрессию с L1 и L2; покажи изменение коэффициентов и метрики, обоснуй выбор.

## Блок 2.4 — Деревья решений и предобработка признаков `[A+B]`

**Зачем:** деревья — основа бустинга; предобработка нужна в любом проекте.

**2.4.1 Дерево решений** `[A]` — разбиения, критерии Gini/энтропия, листья, правила. 🎯 *Докажи себе:* дерево глубины 2 → выведи правила словами.

**2.4.2 Переобучение деревьев** `[A]` — ограничение глубины/листьев, обрезка. 🎯 *Докажи себе:* кривая метрики от максимальной глубины.

**2.4.3 Кодирование категорий** `[A]` — one-hot, label, target encoding и его опасность. 🎯 *Докажи себе:* три способа кодирования, сравни число колонок.

**2.4.4 Масштабирование и пропуски** `[A]` — Standard/MinMax, стратегии заполнения, индикаторы. 🎯 *Докажи себе:* заполни пропуски медианой внутри Pipeline (без утечки).

**2.4.5 Признаки из дат и текста (базово)** `[A]` — компоненты даты, Bag-of-Words/TF-IDF (мост к эмбеддингам). 🎯 *Докажи себе:* из даты сделай «день недели» и «месяц».

**Ресурсы:** ML Handbook (деревья) — RU; mlcourse.ai (деревья и kNN) — RU; Kaggle Learn Intermediate ML — EN.

**Контрольное:** собери полный препроцессинг в `ColumnTransformer`, обучи дерево, убедись через CV в отсутствии утечки.

## Блок 2.5 — Ансамбли и бустинг `[B]`

**Зачем:** бустинг — лучший инструмент для табличных данных и частый вопрос на интервью.

**2.5.1 Идея ансамблей** — bagging vs boosting. 🎯 *Докажи себе:* разница в двух предложениях.

**2.5.2 Random Forest** — бэггинг деревьев, случайность признаков, OOB. 🎯 *Докажи себе:* одно дерево vs RF по CV.

**2.5.3 Градиентный бустинг** — последовательное исправление ошибок. 🎯 *Докажи себе:* что «бустит» каждое следующее дерево.

**2.5.4 XGBoost / LightGBM / CatBoost** — отличия; CatBoost (Яндекс) для категориальных. 🎯 *Докажи себе:* CatBoost на категориях без one-hot vs дерево.

**2.5.5 Тюнинг гиперпараметров** — grid/random search, Optuna, ранняя остановка. 🎯 *Докажи себе:* random search по `learning_rate`/`depth`, зафиксируй прирост.

**Ресурсы:** ML Handbook (ансамбли, бустинг) — RU; mlcourse.ai — RU; **CatBoost** туториалы (`catboost.ai`) — *проверено на июнь 2026*; StatQuest «Gradient Boost / XGBoost» — EN.

**Контрольное:** на Kaggle-датасете обгони линейную baseline бустингом с тюнингом; зафиксируй CV до/после.

## Блок 2.6 — Отбор и инженерия признаков `[B]`

**2.6.1 Важность признаков** — встроенная и permutation. 🎯 *Докажи себе:* топ-10 важных признаков бустинга.

**2.6.2 Методы отбора** — фильтры (корреляция, χ²), обёртки (RFE), встроенные (L1). 🎯 *Докажи себе:* убери половину неважных, проверь метрику.

**2.6.3 Инженерия признаков** — новые признаки, агрегаты, взаимодействия. 🎯 *Докажи себе:* добавь 1 осмысленный признак с приростом.

**2.6.4 Мультиколлинеарность** — почему вредит линейным моделям. 🎯 *Докажи себе:* найди скоррелированную пару через heatmap.

**Ресурсы:** ML Handbook + mlcourse.ai — RU; Воронцов (отбор признаков, презентация 2025) — RU `[B]`.

**Контрольное:** улучши результат 2.5 одним новым признаком и одним раундом отбора; докажи прирост по CV.

## Блок 2.8 — Обучение без учителя `[B]`

**Зачем:** кластеризация и снижение размерности понадобятся в RAG (кластеризация эмбеддингов, визуализация).

**2.8.1 K-means** — алгоритм, выбор k (локоть, силуэт), ограничения. 🎯 *Докажи себе:* кластеризуй 2D + подбери k локтем.

**2.8.2 Иерархическая и DBSCAN** — дендрограмма, плотностная кластеризация. 🎯 *Докажи себе:* набор точек, где DBSCAN лучше K-means.

**2.8.3 PCA** ⭐ — снижение размерности через собственные векторы (связь с 1.1.9–1.1.10), объяснённая дисперсия. 🎯 *Докажи себе:* сожми до 2 компонент, визуализируй.

**2.8.4 t-SNE / UMAP** — визуализация многомерных данных (эмбеддингов). 🎯 *Докажи себе:* визуализируй эмбеддинги через UMAP.

**Ресурсы:** ML Handbook (без учителя, PCA) — RU; mlcourse.ai — RU; Воронцов — RU `[B]`.

**Контрольное:** возьми тексты → TF-IDF эмбеддинги → PCA/UMAP до 2D → покажи, что похожие тексты рядом.
