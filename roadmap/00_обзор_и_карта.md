# Дорожная карта: Frontend → AI Engineer

*Часть 0 из дорожной карты — обзор и карта верхнего уровня.*

## Резюме стратегии

Мы максимально опираемся на инженерный бэкграунд: Python добивается за пару недель, а API, деплой и особенно фронтенд превращаются в конкурентное преимущество — full-stack AI-приложения с нормальным интерфейсом умеют делать единицы ML-щиков. План разведён на два слоя:

- **(А) критический путь до найма** — Python → необходимый минимум математики и ML-грамотности → ядро AI-инженерии (LLM API, RAG, агенты, eval, деплой) → портфолио и собеседования.
- **(Б) слой глубокого понимания** — полная математика с нуля, классический ML с выводом формул и глубокое обучение «руками» (backprop, свой трансформер), дающие понимание изнутри.

Эти цели честно тянут в разные стороны: глубина медленная и для роли AI Engineer по большей части не обязательна — поэтому слой Б не вырезаем, но ведём параллельно и часть откладываем на «после найма», а слой А двигаем агрессивно вперёд.

**Реалистичная оценка при ~20–25 ч/нед:**
- до состояния «можно подаваться на AI Engineer» (приоритет слоя А) — **~7–9 месяцев**;
- полная программа со слоем Б — **~12–15 месяцев** (часть Б продолжается уже в роли).

---

## Легенда

- **[A]** — критический путь до найма (обязательный минимум).
- **[B]** — слой глубокого понимания (параллельно или «после найма»).
- **[A+B]** — у блока есть минимальный срез для А и полная глубина для Б.

## Карта верхнего уровня: этапы → блоки

### ПРЕДЭТАП (старт) — SQL для AI Engineer · ~8–15 ч · [A]
- Рабочий SQL: выборки, агрегаты, JOIN, CTE; PostgreSQL + pgvector; text-to-SQL.
- **Делается первым** — совпадает с месяцем плотного SQL в Яндекс.Практикуме. Детали: `10_sql_для_ai_engineer.md`.

### ЭТАП 0 — Python и инструментарий ML · ~2–3 нед · [A]
- 0.1 От JS/TS к Python: синтаксис, типы, структуры данных, ООП, асинхронность, особенности vs JS.
- 0.2 Научный стек: NumPy (векторизация), Pandas (таблицы), Matplotlib/Seaborn (графики).
- 0.3 Среда: Jupyter/Colab/Kaggle, venv/conda, Git под ML-проекты, базовый CLI.
- 0.4 Твоё железо: когда Mac (MPS), когда облако (Colab/Kaggle), где понадобится платная GPU.

### ЭТАП 1 — Математический фундамент с нуля · полный ~12–16 нед [B] / минимум-для-ML ~5–6 нед [A]
- 1.0 Предполёт: математическая нотация, функции, степени/логарифмы, Σ/∏, графики. [A+B]
- 1.1 Линейная алгебра (векторы, матрицы, собственные значения и т.д.). [A+B]
- 1.2 Математический анализ (производные, градиенты, цепное правило). [A+B]
- 1.3 Теория вероятностей. [A+B]
- 1.4 Статистика (оценки, распределения, проверка гипотез). [A+B]

### ЭТАП 2 — Классический ML · полный ~6–8 нед [B] / ядро ~3–4 нед [A]
- 2.1 Рабочий процесс и постановка задачи: train/val/test, переобучение, утечки. [A]
- 2.2 Метрики качества (классификация, регрессия, ранжирование). [A]
- 2.3 Линейные модели: линейная и логистическая регрессия, градиентный спуск, регуляризация. [A+B]
- 2.4 Деревья решений и предобработка признаков. [A+B]
- 2.5 Ансамбли и бустинг (Random Forest, Gradient Boosting, XGBoost/LightGBM/CatBoost). [B]
- 2.6 Отбор и инженерия признаков. [B]
- 2.7 Интерпретация моделей (feature importance, SHAP). [B]
- 2.8 Обучение без учителя (кластеризация, PCA, снижение размерности). [B]

### ЭТАП 3 — Глубокое обучение изнутри · полный ~8–10 нед [B] / A-срез ~3–4 нед [A]
- 3.1 Нейросети с нуля: перцептрон, MLP, активации, прямой проход. [B]
- 3.2 Обратное распространение: backprop руками, автоград, оптимизаторы. [B]
- 3.3 PyTorch: тензоры, autograd, nn.Module, обучающий цикл, MPS на Mac. [A+B]
- 3.4 Свёрточные сети (CNN) — обзорно, для общей картины. [B]
- 3.5 Рекуррентные сети (RNN/LSTM/GRU) и последовательности. [B]
- 3.6 Внимание (attention) и трансформеры — построение руками. [A+B]
- 3.7 Токенизация, эмбеддинги, языковые модели — мост к LLM. [A]

### ЭТАП 4 — AI-инженерия: LLM-приложения · ~8–10 нед · [A] (ядро критического пути)
- 4.1 LLM API: модели, токены, параметры, стриминг, структурированный вывод. [A]
- 4.2 Промт-инжиниринг: системные промпты, few-shot, reasoning-паттерны. [A]
- 4.3 Эмбеддинги и семантический поиск. [A]
- 4.4 Векторные базы данных. [A]
- 4.5 RAG (Retrieval-Augmented Generation). [A]
- 4.6 Агенты и tool use / function calling. [A]
- 4.7 MCP (Model Context Protocol). [A]
- 4.8 Оценка качества (eval) LLM-приложений. [A]
- 4.9 Фреймворки и оркестрация (LangChain/LlamaIndex и альтернативы). [A]
- 4.10 (опционально) Дообучение: fine-tuning, LoRA/QLoRA. [B]

### ЭТАП 5 — Инференс, деплой, продакшн и выход на роль · ~6–8 нед · [A]
- 5.1 Инференс и оптимизация: квантизация, локальный (Ollama/MPS) vs hosted inference, vLLM. [A]
- 5.2 Деплой full-stack AI-приложения (преимущество: фронтенд + API). [A]
- 5.3 Наблюдаемость, стоимость, безопасность (observability, guardrails, prompt injection, бюджет). [A]
- 5.4 Основы MLOps по необходимости: эксперименты, версионирование, CI/CD. [A]
- 5.5 Портфолио: 3–4 проекта. [A]
- 5.6 Чек-лист компетенций AI Engineer + подготовка к собеседованию. [A]
- 5.7 Поиск работы и позиционирование фронтенд-бэкграунда. [A]

## Как это параллелится (важно для скорости)

Слой А не обязан ждать полного слоя Б:

0. **Старт — SQL** (предэтап): идёт первым, параллельно месяцу SQL в Яндекс.Практикуме. Закрывает базу и pgvector-задел под RAG.
1. Проходишь **Этап 0** целиком (он нужен всем и быстрый).
2. Берёшь **минимальный срез Этапа 1** (1.0 + ключевое из 1.1–1.2) и **ядро Этапа 2** (2.1–2.3).
3. После этого **стартуешь Этап 4 параллельно**, не дожидаясь полного прохождения Этапов 1–3. Этап 4 + Этап 5 — кратчайший путь к собеседованиям.
4. Полную глубину Этапов 1–3 (слой Б) ведёшь фоновым треком — часть до найма, часть после.
