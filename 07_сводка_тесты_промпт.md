## 3. Мини-промпт для генерации проверочного задания

Если по подтеме нет готового теста — вставь этот промпт в любую сильную LLM, подставив тему. Он спроектирован так, чтобы выдать **одно конкретное действие** и **критерий самопроверки**, а не абстракции, и не выдать решение сразу.

```
Ты — придирчивый наставник по ML/AI-инженерии. Я — senior frontend-разработчик
(8 лет, JS/TS/React), осваиваю ML с нуля. Сейчас проверяю усвоение темы:

ТЕМА: «<вставь подтему, например: косинусная близость / RAG re-ranking / backprop>»
МОЙ УРОВЕНЬ ПО ТЕМЕ: <новичок / средний / уверенный>
ФОРМАТ: <практика на Python-NumPy/PyTorch / теоретическое объяснение / разбор кода>

Сгенерируй РОВНО ОДНО проверочное задание по этой теме:
1. Конкретное и выполнимое за 15–40 минут, без воды.
2. Если уместно — практическое (код, который реально надо написать и запустить),
   а не «расскажи своими словами».
3. Калибруй сложность под мой уровень и опыт инженера (базу программирования
   можно не объяснять).
4. Дай чёткий КРИТЕРИЙ УСПЕХА: как я пойму, что выполнил верно
   (ожидаемый результат / свойство / число для сверки).
5. Решение НЕ показывай сразу — помести его и типичные ошибки в самый конец
   под заголовком «=== ОТВЕТ (не подглядывать) ===».

Если тема позволяет, добавь один короткий «каверзный» вопрос на понимание,
который ловит иллюзию знания.
```

Вариант для теории (когда код не нужен): замени пункт ФОРМАТ на `теоретическое объяснение` — тогда задание будет «объясни X так, чтобы поймать пробелы», с критерием, какие именно моменты обязательно должны прозвучать.
