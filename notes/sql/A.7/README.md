# A.7 — Описательная статистика в SQL (перцентили, разброс, центр)

> Дистиллят заметок Практикума (описательная статистика и аналитические функции).

## Суть простыми словами
Описать большой набор чисел парой величин: где его «центр» и насколько данные разбросаны. В SQL для этого есть готовые функции.

## Ключевое

### Тип величины
- **Количественная** — можно складывать (выручка, число заказов). **Категориальная** — сложение бессмысленно (номер группы, телефон). Тест: складываются два значения осмысленно → количественная.

### Центр (меры центральной тенденции)
- **Мода** — самое частое значение (плоха, если все встречаются поровну).
- **Медиана** — центральное значение отсортированного ряда (устойчива к выбросам).
- **Среднее** — учитывает каждое значение (чувствительно к выбросам вроде `100` среди `1,2,5,8`).

### Разброс
- **Размах** = max − min (годится, когда нет выбросов).
- **Дисперсия** = средний квадрат отклонений от среднего.
- **Стандартное отклонение** = √дисперсии; описывает «типичный» разброс вокруг среднего (среднее 7, σ≈5 → большинство значений в 2…12).

### Функции SQL
- `PERCENTILE_DISC(p) WITHIN GROUP (ORDER BY col)` — перцентиль, возвращает **реальное** значение из данных.
- `PERCENTILE_CONT(p)` — интерполирует между соседними (может дать дробное). На малых выборках (<100 строк) `DISC` неточен → бери `CONT`.
- `STDDEV(col)` — стандартное отклонение; работает и как оконная: `STDDEV(col) OVER (PARTITION BY region)`.
- Квартили: 25% / 50% (= медиана) / 75%. Перцентили/квартили — описательные статистики, но **не** меры центра.

## Пример
```sql
SELECT PERCENTILE_DISC(0.15) WITHIN GROUP (ORDER BY session_duration)
FROM sessions;                       -- 15-й перцентиль

SELECT region,
       STDDEV(session_duration) OVER (PARTITION BY region) AS sd
FROM sessions;
```

## Грабли
- `PERCENTILE_DISC` на малой выборке округляет к реальному значению → бери `PERCENTILE_CONT`.
- Среднее «врёт» при выбросах — тогда смотри медиану.
