# A.3 — Дата и время

> Дистиллят заметок Практикума (типы и операторы для даты/времени в PostgreSQL).

## Суть простыми словами
Дата/время — отдельные типы (не строки!). Приведи строку к типу → и можно вычитать даты, прибавлять интервалы, выделять части (месяц, день недели).

## Ключевое

### Типы
- `timestamp` — дата + время. Две версии: `timestamp without time zone` (по умолчанию) и `timestamp with time zone` (= `timestamptz`, часовой пояс по UTC).
- `date` — только дата (при приведении к `date` **время теряется**). `time` — только время.
- `interval` — разница двух дат: `'2 weeks 1 day 36 minutes'`. Интервалы можно складывать/вычитать.

### Операции
- Строки вычитать нельзя → приводи: `'2010-07-08'::timestamp - '2010-06-30'::timestamp`.
- `EXTRACT(MONTH FROM дата::timestamp)` — выделить часть. Части: `YEAR/QUARTER/MONTH/WEEK/DAY/HOUR/...`, `DOW` (день недели 0=вс), `ISODOW` (1=пн…7=вс), `DOY` (день года).
- `AGE(конец, начало)` — разница в человекочитаемом виде (`5 months 29 days`).
- Прибавить срок: `CAST('20-03-2024' AS date) + '5 months'::interval`.
- `to_timestamp(1714521600)` — из Unix-времени (секунды с 1970-01-01 UTC) в `timestamp`.

## Пример
```sql
SELECT EXTRACT(MONTH FROM CAST('1983-05-15' AS timestamp));   -- 5
SELECT CURRENT_DATE - MAX(created_at) AS days_since_last_order FROM orders;
```

## Грабли
- Вычитание/сравнение строк-дат → ошибка; всегда `::timestamp`/`::date`.
- Приведение к `date` молча выкидывает время.
- `DOW` считает воскресенье за 0, `ISODOW` — за 7. Легко ошибиться на день.
