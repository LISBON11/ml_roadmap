# A.4 — Оконные функции: OVER, PARTITION BY, ORDER BY

> Дистиллят заметок Практикума (агрегирующие оконные функции).

## Суть простыми словами
Оконная функция считает по группе строк («окно»), но **сохраняет все строки** и дописывает результат новым столбцом. Это ключевое отличие от `GROUP BY`, который схлопывает группу в одну строку.

## Ключевое
- Синтаксис: `<функция>(...) OVER (PARTITION BY ... ORDER BY ...)`.
- `PARTITION BY` — делит на окна (аналог групп), но строки не схлопывает. `ORDER BY` внутри `OVER()` — упорядочивает внутри окна.
- **Без `ORDER BY`** в `OVER()`: агрегат по всему окну целиком (одно и то же число во всех строках окна).
- **С `ORDER BY`** в `OVER()`: **накопительный итог** (от начала окна до текущей строки) — кумулятивная сумма/среднее.
- `OVER ()` пустой → окно = вся таблица.
- Сочетается с `ROUND()`, `AVG`, `SUM`, `COUNT`, `STDDEV` и т.д.

|  | `PARTITION BY` | `GROUP BY` |
|---|---|---|
| Строки | сохраняет все | схлопывает в одну на группу |
| Зачем | оконные функции | агрегаты по группам |

### Грабля с WHERE
Оконные функции считаются **после** `WHERE`/`GROUP BY` → отфильтровать их результат в том же запросе нельзя. Оборачивай в подзапрос/CTE и фильтруй снаружи.

## Пример
```sql
-- кумулятивная выручка по дням
SELECT event_dt, revenue,
       SUM(revenue) OVER (ORDER BY event_dt) AS cumulative_revenue
FROM orders;

-- сумма за каждый день (без схлопывания строк)
SELECT event_dt, revenue,
       SUM(revenue) OVER (PARTITION BY DATE_TRUNC('day', event_dt)) AS daily_revenue
FROM orders;

-- фильтр по оконному значению — через подзапрос
SELECT * FROM (
    SELECT user_id, session_duration,
           AVG(session_duration) OVER () AS avg_dur
    FROM sessions
) t
WHERE session_duration > avg_dur;
```

## Грабли
- Нельзя фильтровать оконный результат в `WHERE` того же уровня → подзапрос/CTE.
- Забыть `ORDER BY` в `OVER()`, когда нужен именно накопительный итог (получишь общую сумму во всех строках).
