# A.5 — Ранжирование: ROW_NUMBER / RANK / DENSE_RANK / NTILE

> Дистиллят заметок Практикума (ранжирующие оконные функции).

## Суть простыми словами
Три способа пронумеровать строки по порядку — отличаются поведением на **одинаковых** значениях (ничьих).

## Ключевое
- `ROW_NUMBER()` — сплошная нумерация 1,2,3… **без учёта ничьих** (одинаковым значениям — разные номера). Аргумента нет.
- `RANK()` — «спортивный» ранг: ничьи получают **одинаковый** ранг, дальше идёт **пропуск** (1,1,3,3,3,**9**…).
- `DENSE_RANK()` — как `RANK()`, но **без пропусков** (1,1,2,2,2,3…).
- `NTILE(n)` — делит строки на `n` примерно равных групп (квартили при `NTILE(4)`).
- Порядок задаётся через `OVER(ORDER BY ...)`. Без `ORDER BY` в окне `ROW_NUMBER()` нумерует в исходном порядке таблицы, **игнорируя** внешний `ORDER BY`.

| значение | ROW_NUMBER | RANK | DENSE_RANK |
|---|---|---|---|
| 8 | 1 | 1 | 1 |
| 8 | 2 | 1 | 1 |
| 7 | 3 | 3 | 2 |
| 6 | 4 | 4 | 3 |

## Пример
```sql
SELECT user_id,
       DENSE_RANK() OVER (ORDER BY COUNT(revenue) DESC) AS users_rank
FROM orders
GROUP BY user_id
ORDER BY users_rank;
```

## Грабли
- Чтобы ранг совпал с сортировкой вывода — сортировку клади **внутрь** `OVER(ORDER BY ...)`, а не только во внешний `ORDER BY`.
- Путаница RANK vs DENSE_RANK: RANK пропускает номера после ничьей, DENSE_RANK — нет.
