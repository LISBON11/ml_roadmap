# A.1 — Множества, подзапросы, CTE

> Дистиллят заметок Практикума (операции множеств и подзапросы).

## Суть простыми словами
Три способа комбинировать запросы: **множества** (склеить/вычесть результаты по вертикали), **подзапрос** (запрос внутри запроса), **CTE** (`WITH` — именованный подзапрос, вынесенный наверх для читаемости и переиспользования).

## Ключевое

### Множества (вертикально, столбцы должны совпадать по числу/типу)
- `UNION` — объединение без дублей; `UNION ALL` — с дублями (быстрее).
- `INTERSECT` — пересечение; `EXCEPT` — вычитание.
- Имена полей в результате берутся из **первого** запроса.

### Подзапросы
- В `WHERE`: **не** давать псевдоним; результат — одно поле. Одно значение → **скалярный**, много → **многострочный** (`IN`, `> ANY`…).
- В `SELECT`: тоже одно поле; обычно коррелированный (условие связи с внешним запросом), псевдоним по желанию.
- Подзапрос vs JOIN: подзапрос отсекает лишнее **до** основной работы (меньше данных); JOIN больших таблиц раздувает результат и грузит БД.

### CTE (`WITH`)
- Выносит подзапрос наверх, даёт имя, позволяет **переиспользовать** результат несколько раз (посчитал среднее один раз — применил в трёх местах).
- Обращаться можно после `FROM`/`JOIN`, **нельзя** после `WHERE` (ограничение PostgreSQL).

## Пример
```sql
WITH avg_clients AS (
    SELECT AVG(cnt) AS a FROM ...
)
SELECT ... FROM orders, avg_clients WHERE ...;

SELECT 'Январь' AS month, 512 AS orders
UNION
SELECT 'Февраль', 473;
```

## Грабли
- `UNION` убирает дубли (сортирует → медленнее); если дубли не мешают — `UNION ALL`.
- Псевдоним у подзапроса в `WHERE` → ошибка.
- Подзапрос в `WHERE`/`SELECT` с более чем одним полем → ошибка.
