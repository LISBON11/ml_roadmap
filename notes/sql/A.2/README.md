# A.2 — Категоризация, вычисляемые столбцы, работа с NULL/дубликатами

> Дистиллят заметок Практикума (категоризация значений, пропуски и дубликаты). Тема Практикума — «Создание новых столбцов», сюда же логично легла чистка данных в SQL.

## Суть простыми словами
`CASE` — SQL-аналог `if/elif/else`: строит новый столбец из условий. `COALESCE` — частный случай для замены `NULL`. Отдельно — как SQL ведёт себя с `NULL` в агрегатах и как ловить дубликаты.

## Ключевое

### Категоризация
- `CASE WHEN условие THEN ... [WHEN ...] ELSE ... END` — новый столбец по правилам. Универсален: заменяет **любые** значения.
- `COALESCE(field, 0)` — вернёт первое не-`NULL`; это частный случай `CASE WHEN field IS NULL THEN 0 ELSE field END`. `COALESCE` всегда можно переписать через `CASE`, наоборот — нет.

### NULL
- Поиск: `WHERE ... IS NULL` / `IS NOT NULL` (не `= NULL`!).
- `COUNT(*)` считает **все** строки (NULL не влияет). `COUNT(col)`, `AVG`, `SUM` **игнорируют** NULL. Для `(4,4,4,4,NULL)`: `AVG=4`, `SUM=16`, `COUNT(col)=4`.
- Тактика: часто NULL **оставляют** — это информация о качестве данных; замена/удаление могут исказить выводы.

### Дубликаты
- Явные дубли убирают группировкой по всем столбцам (`GROUP BY *`), но это «тяжёлая» операция по памяти.

### Грабля с делением
- `COUNT(discount) / COUNT(*)` → **0**: деление целых даёт целую часть (`5/2 = 2`). Приведи тип: `COUNT(discount) / CAST(COUNT(*) AS real)`.

## Пример
```sql
SELECT COALESCE(discount, 0) AS discount FROM orders;

SELECT COUNT(*) FROM orders WHERE discount IS NULL;   -- сколько пропусков
SELECT COUNT(discount) / CAST(COUNT(*) AS real) AS filled_share FROM orders;
```

## Грабли
- `= NULL` не работает — только `IS NULL`.
- Деление двух целых обрезает дробь → приведи одно к `real`/`numeric`.
- Агрегаты по столбцу молча пропускают NULL — легко получить смещённое среднее.
