# Задачи на область видимости (scope) — разбор + тренажёр

> Приложение к [README](README.md). Часть A — разбор базовых 10 (с ответами). Часть B — 10 посложнее (ответы в самом низу, сначала попробуй сам).

## Два правила, закрывающих большинство ловушек
1. **Присваивание имени где угодно в функции** (`=`, `+=`, `for x in ...`, даже под `if False`, даже `del x`) делает имя **локальным на всю функцию**. Чтение до присваивания → `UnboundLocalError`. Scope определяется статически (по тексту), до запуска.
2. **Изменяемый аргумент по умолчанию** (`def f(a, box=[])`) создаётся **один раз** и копит состояние между вызовами.

Плюс: **мутация ≠ переприсваивание** (`lst.append(x)` не требует `global`, `lst = [...]` — требует). В Python нет `undefined`: пустой локальный слот → ошибка, а не значение (как `let` в JS, не `var`).

## Часть A. Разбор базовых 10

### 1 — `UnboundLocalError`
```python
x = 10
def f():
    x = x + 5      # x локальна (есть присваивание) → читаем до создания
    return x
print(f())
```
`x + 5` читает локальную `x`, которой ещё нет. Правило 1.

### 2 — `UnboundLocalError` (не 10!)
```python
x = 10
def f():
    if False:
        x = 99     # никогда не выполнится, НО присутствует в тексте → x локальна
    return x
print(f())
```
Scope считается статически: важно наличие строки `x = 99`, а не то, выполнилась ли она. `return x` читает пустую локальную.

### 3 — `1 2 2`
```python
count = 0
def tick():
    global count
    count += 1
    return count
print(tick(), tick(), count)
```
`global` разрешает менять модульную `count`. Аргументы `print` считаются слева направо: `tick()`→1, `tick()`→2, `count`=2.

### 4 — `[2, 2, 2]` (позднее связывание)
```python
funcs = []
for i in range(3):
    funcs.append(lambda: i)     # все замыкают ПЕРЕМЕННУЮ i, не значение
print([g() for g in funcs])
```
После цикла `i = 2` (range(3) = 0,1,2). Все лямбды читают одну и ту же `i` → `2, 2, 2`. Классический баг «late binding».

### 5 — `[0, 1, 2]` (фикс задачи 4)
```python
funcs = []
for i in range(3):
    funcs.append(lambda i=i: i)  # значение i заморожено в аргументе по умолчанию
print([g() for g in funcs])
```
`i=i` вычисляется **в момент создания** лямбды → каждая хранит своё значение.

### 6 — `[1]`, `[1, 2]`, `[3]`, `[1, 2, 4]`
```python
def add(item, box=[]):
    box.append(item)
    return box
print(add(1))       # [1]        — дефолтный список
print(add(2))       # [1, 2]     — ТОТ ЖE список
print(add(3, []))   # [3]        — передали свой
print(add(4))       # [1, 2, 4]  — снова дефолтный, он копил
```
Правило 2. Фикс: `box=None` + `if box is None: box = []`.

### 7 — `[1, 2, 3]`
```python
data = [1, 2]
def mutate():
    data.append(3)   # мутация — global не нужен
def rebind():
    data = [9, 9]    # локальная, внешний data не тронут
mutate(); rebind()
print(data)
```
Мутация меняет внешний объект; переприсваивание создаёт локальную.

### 8 — `"changed"`
```python
def outer():
    x = "enclosing"
    def inner():
        nonlocal x   # писать в переменную объемлющей функции
        x = "changed"
    inner()
    return x
print(outer())
```
`nonlocal` направляет присваивание в `x` из `outer`.

### 9 — `enclosing changed`
```python
x = "global"
def outer():
    x = "enclosing"
    def inner():
        global x     # тянется к МОДУЛЮ, минуя outer
        x = "changed"
    inner()
    return x
print(outer(), x)
```
`outer()` вернёт свою `"enclosing"` (не тронута), а `global x` изменит модульную → второй `x` = `"changed"`.

### 10 — `UnboundLocalError`
```python
n = 5
def f():
    print(n)             # читаем n до её создания
    for n in range(3):   # for — это присваивание n → n локальна во всей f
        pass
f()
```
Тот же принцип, что в №2, но присваивание спрятано в `for`.

### 11.

```python
def make_multipliers():
    result = []
    for x in range(1, 4):
        def mult(n):
            return n * x
        result.append(mult)
    return result
```

### 12.

```python
def counter(fn):
    calls = 0
    def wrapper(*args, **kwargs):
        nonlocal calls
        calls += 1
        return f"#{calls}: {fn(*args, **kwargs)}"
    return wrapper

@counter
def hi():
    return "hi"
print(hi()); print(hi())
```

### 13.

### 15.

```python
def a():
```

## Часть C. Ответы к 11–20

<details>
<summary>Показать ответы</summary>

**11 → `30 30 30`.** Позднее связывание, но уже во вложенном `def` (не только в lambda): все `mult` замыкают переменную `x`, после цикла `x = 3`. Фикс — заморозить: `def mult(n, x=x):`.

**12 → `#1: hi`, затем `#2: hi`.** Декоратор — это замыкание; `nonlocal calls` хранит состояние между вызовами `wrapper`. Так пишут счётчики вызовов, кэш, rate-limit.

**13 → `NameError: name 'base' is not defined`.** Тело comprehension выполняется в **своей** функциональной области, и она **не видит** переменные класса (класс-scope не входит в цепочку enclosing). Исключение — самый левый итерабельный (`range(3)`), он считается в scope класса. Обход: `scaled = [base + i for base in [base] for i in range(3)]` или вынести в `__init__`.

**14 → `before`.** В Python 3 comprehension имеет собственную область — переменная цикла `x` **не утекает** наружу (в Python 2 утекала бы → `2`).

**15 → `99`.** `nonlocal x` в `c` ищет ближайшую объемлющую функцию, где `x` **связана**: в `b` её нет, в `a` — есть. Значит меняется `x` из `a`.

**16 → `42`.** `global` может **создать** переменную уровня модуля, даже если её не было до вызова.

**17 → `{'a': 0}`, затем `{'a': 0, 'b': 1}`.** Тот же mutable-default, что и со списком: словарь по умолчанию один на все вызовы, `len(db)` растёт.

**18 → `('inner', 'outer') global`.** `inner` завела свою локальную `x` (нет `nonlocal`) → вернула `"inner"`, но `x` в `outer` осталась `"outer"`. Модульная `x` не тронута → `"global"`.

**19 → `UnboundLocalError`.** `del x` удаляет связывание, слот снова пуст (но `x` всё ещё локальна). `print(x)` читает пустую локальную.

**20 → `UnboundLocalError`.** `nums += [4]` — это присваивание `nums` (target слева) → `nums` локальна. `+=` должен сначала **прочитать** локальную `nums` → её ещё нет. Коварно: `nums.append(4)` сработал бы (чистая мутация, без присваивания имени), а `+=` — нет. Фикс: `global nums`.

</details>
