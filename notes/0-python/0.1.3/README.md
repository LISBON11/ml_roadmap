# 0.1.3 — Функции: область видимости (scope), global, nonlocal

> Как Python решает, к какой переменной относится имя внутри функции. Родственные темы: замыкания, [[0.1.2]] (изменяемые/неизменяемые объекты — важно для «грабель» ниже).

## Суть простыми словами
Функция видит внешние переменные на **чтение**, но при **присваивании** (`x = ...`) по умолчанию заводит **свою локальную** — внешняя не меняется. Чтобы писать во внешнюю, нужно явно сказать `global` (для модульной) или `nonlocal` (для объемлющей функции).

## Ключевое

### Правило LEGB — где Python ищет имя
Поиск идёт изнутри наружу, первое совпадение выигрывает:
- **L**ocal — локальные переменные текущей функции;
- **E**nclosing — объемлющая функция (для вложенных);
- **G**lobal — уровень модуля (файла);
- **B**uilt-in — встроенные (`len`, `sum`, `print`…).

### Присваивание → локальная (главное правило)
- Если в функции **где-либо** есть `x = ...`, Python на **всём** протяжении функции считает `x` локальной.
- Поэтому чтение до присваивания → `UnboundLocalError` (не значение снаружи!).

```python
value = 100
def f():
    print(value)      # UnboundLocalError:
    value = 200       # из-за этой строки value локальна во всей f
```

### Чтение внешней — можно без объявлений
```python
value = 100
def show():
    print(value)      # 100 — читаем глобальную, всё ок
```

### `global` — писать в глобальную (модульную)
```python
value = 100
def f():
    global value
    value = 200       # меняет внешнюю
f(); print(value)     # 200
```

### `nonlocal` — писать в объемлющую функцию (замыкания)
```python
def counter():
    n = 0
    def inc():
        nonlocal n    # без него n = n + 1 → UnboundLocalError
        n += 1
        return n
    return inc

c = counter()
c(); c()              # 1, 2 — состояние живёт в замыкании
```
`global` тянется к модулю **через любую вложенность**; `nonlocal` — только к ближайшей объемлющей функции (не к глобальной).

### Мутация ≠ присваивание (частая путаница)
Менять **содержимое** изменяемого объекта (list/dict) можно без `global` — это не присваивание имени:
```python
data = []
def add():
    data.append(1)    # ок, global НЕ нужен — мы не переприсваиваем имя data
def reset():
    data = []         # а это локальная! внешний data не тронут (нужен global)
```

## Грабли
- `UnboundLocalError` = «присвоил ниже → имя стало локальным → прочитал до присвоения». Лечится `global`/`nonlocal` или чтением без присваивания.
- `x += 1` — это тоже присваивание → та же ловушка (нужен `global`/`nonlocal`).
- `list.append()`/`dict[k]=v` работают без `global`, а `list = [...]` — нет. Мутация vs переприсваивание.
- **Позднее связывание в циклах:** `lambda: i` внутри цикла запомнит *переменную* `i`, а не её значение → все замыкания вернут последнее `i`. Фикс: `lambda i=i: i` (значение в дефолт аргумента).
- **Изменяемый аргумент по умолчанию:** `def f(a, acc=[])` — `acc` создаётся **один раз** и переиспользуется между вызовами. Фикс: `acc=None` + `if acc is None: acc = []`.

## Пример / код
```python
# счётчик без global — через nonlocal и замыкание
def make_counter(start=0):
    n = start
    def step(k=1):
        nonlocal n
        n += k
        return n
    return step
```

## Ссылки
- Best practice: избегай `global` в своём коде — возвращай значение и присваивай снаружи (чистые функции, вход→выход). `global`/`nonlocal` держи для чтения чужого кода и для счётчиков-замыканий.
