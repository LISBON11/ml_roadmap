# Включения (comprehensions) и вложенные списки

> Приложение к [0.1.2](README.md). Компактное создание коллекций одной строкой + грабля со списком списков.

## Списковое включение (list comprehension)
Компактно построить новый список из другого итерируемого объекта.

Форма: `[func(x) for x in iterable if filter]` — четыре части:
- `iterable` — что перебираем (список, строка, `range()`…);
- `x` — переменная, куда попадает элемент;
- `func(x)` — преобразование элемента (`x**2`, `x.upper()`);
- `if filter` — необязательный фильтр (оставить не все).

```python
[x for x in range(10)]                       # 0..9
[x**2 for x in numbers]                       # квадраты
[c.upper() for c in phrase if is_vowel(c)]    # только гласные, в верхнем регистре
```

## Словарное включение (dict comprehension)
То же, но результат — словарь; задаёшь и ключ, и значение:
`{key_expr: value_expr for item in iterable if condition}`

```python
{w: len(w) for w in words}                    # слово → его длина
```

## Вложенные структуры (списки списков)
Матрицу/таблицу хранят как список списков; доступ по двум индексам: `m[строка][столбец]`, `m[-1][-1]` — последний элемент последней строки.

### ⚠️ Главная грабля: `[[...]] * n` копирует ССЫЛКУ
```python
grid = [[0, 0, 0]] * 3        # три ССЫЛКИ на ОДИН и тот же список
grid[-1][-1] = 5
# [[0, 0, 5], [0, 0, 5], [0, 0, 5]]  ← изменились ВСЕ строки!
```
`* 3` не создаёт три независимых списка — он трижды повторяет **один** объект в памяти. Меняешь одну «строку» — меняются все.

### Правильно — каждый раз новый список
```python
grid = [[0, 0, 0] for _ in range(3)]          # включение создаёт новый список на каждой итерации
grid = [[0 for _ in range(N)] for _ in range(M)]   # N×M произвольного размера

# или циклом
grid = []
for _ in range(3):
    grid.append([0, 0, 0])
```

## Грабли
- `[[0]*3]*3` (и `[[...]]*n` вообще) — общие ссылки на один вложенный список. Для независимых строк — включение `[[...] for _ in range(n)]`.
- Это та же природа, что «мутация ≠ копия»: `*` дублирует ссылку, а не содержимое (см. изменяемые объекты в [0.1.2](README.md)).
- `_` в `for _ in range(n)` — соглашение «переменная не нужна».

## Докажи себе
- [ ] Воспроизвёл(а) баг `[[0,0,0]]*3` (меняется вся «колонка») и починил(а) включением.
- [ ] Собрал(а) матрицу N×M нулей и словарь `{слово: длина}` включением.
