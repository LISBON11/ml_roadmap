# Форматирование в f-строках — спецификатор `{значение:формат}`

> Из вопроса про `f"{cpc:.2f}"`. Двоеточие в f-строке отделяет **что** вывести от того, **как** это показать. Меняет только вывод (строку) — само число не округляется.

## Суть простыми словами
`{значение:формат}` — после `:` идёт мини-язык формата: сколько знаков, разделители тысяч, проценты, ширина, выравнивание. Ближайший аналог из JS — `num.toFixed(2)`, только в Python формат пишется прямо в строке и умеет больше.

## Разбор `{cpc:.2f}`
| часть | значение |
|-------|----------|
| `cpc` | что вывести |
| `:` | «дальше формат» |
| `.2` | 2 знака после точки |
| `f` | fixed-point, обычное десятичное |

```python
cpc = 15000.50 / 300
print(f"{cpc:.2f}")   # 50.00  ← округлён вывод
print(f"{cpc}")       # 50.001666666666665  ← без формата
```

## Частые форматы (для аналитики)
```python
x = 1234567.891
f"{x:.2f}"        # '1234567.89'      — 2 знака после точки
f"{x:,.2f}"       # '1,234,567.89'    — разделитель тысяч
f"{x:_.2f}"       # '1_234_567.89'    — разделитель через _
f"{0.1875:.1%}"   # '18.8%'           — проценты (×100 и знак %)
f"{42:03d}"       # '042'             — целое, дополнить нулями до 3
f"{255:x}"        # 'ff'              — шестнадцатеричное
f"{x:>12.2f}"     # '  1234567.89'    — вправо в ширину 12 (`<` влево, `^` по центру)
```

## Мостик из JS
```js
`${cpc.toFixed(2)} руб.`   // 50.00
```
Разница: в JS формат — **метод у числа** (`.toFixed`, `.toLocaleString`); в Python — **мини-язык** после `:` (`,`/`%`/ширина/выравнивание в одном месте).

## Грабли
- Форматирование меняет **только строку вывода**, исходное число не трогает.
- `.2f` **округляет** (не отбрасывает): `f"{2.005:.2f}"` может дать `2.00` из-за двоичного представления float — для денег критично считать в `Decimal`.
- Тип обязателен под данные: `d` — только для int (`f"{3.5:d}"` → ошибка), `f`/`%` — для чисел, не для строк.
