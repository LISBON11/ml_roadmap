## Шаг 2 — Чистка пропусков (45 мин)
Сначала посмотри масштаб:
```python
df.isna().sum().sort_values(ascending=False)
```
Стратегия (думай, а не делай механически):
- Колонка, где пропусков **очень много** (в Titanic — `Cabin`) — часто проще удалить колонку или превратить в признак «есть/нет каюты».
- Числовая с умеренными пропусками (`Age`) — заполни **медианой** (устойчивее к выбросам, чем среднее — это из статистики недели 6).
- Категориальная с парой пропусков (`Embarked`) — заполни самым частым значением (модой).

```python
df["Age"] = df["Age"].fillna(df["Age"].median())
df["Embarked"] = df["Embarked"].fillna(df["Embarked"].mode()[0])
df["HasCabin"] = df["Cabin"].notna().astype(int)   # пример инженерии признака
df = df.drop(columns=["Cabin"])
```

🎯 *Докажи себе:* после чистки `df.isna().sum()` по используемым колонкам = 0, и ты можешь объяснить, **почему** для `Age` выбрал медиану.

> ⚠️ Запомни на будущее (Этап 2.1.5): когда дойдёт до моделей, заполнять пропуски надо **после** train/test-сплита, иначе утечка. Здесь, в чистом EDA, это ещё не критично — но привыкай к мысли.

## Шаг 3 — Разведка: задаём данным вопросы (60 мин)
Хорошая EDA — это не графики ради графиков, а **вопросы и ответы**. Сформулируй 3–4 вопроса и ответь на них через `groupby`.

Примеры вопросов к Titanic:
```python
# Выживаемость по полу
df.groupby("Sex")["Survived"].mean()

# Выживаемость по классу каюты
df.groupby("Pclass")["Survived"].mean()

# Связка: пол + класс
df.groupby(["Sex", "Pclass"])["Survived"].mean()
```

🎯 *Докажи себе:* можешь словами сформулировать 3 наблюдения из чисел (например, «женщины выживали заметно чаще»; «1-й класс выживал чаще 3-го»).

# 5. Возраст vs выживаемость по полу (boxplot)
