## Блок 0.1 — От JS/TS к Python

**Зачем:** Python — рабочий язык всего ML; нужно не «выучить язык», а быстро переложить навыки и добрать питон-специфику.

### Подтемы

**0.1.1 Синтаксис и базовые типы** ⚡
- Отступы вместо `{}`, отсутствие `;`, `snake_case` против `camelCase`
- `int` / `float` / `str` / `bool` / `None`; динамическая типизация
- Операторы, тернарник (`a if cond else b`), `is` vs `==`, truthy/falsy
- f-строки и форматирование
- 🎯 *Докажи себе:* перепиши любую свою маленькую JS-утилиту на Python.

**0.1.2 Структуры данных** ⚡
- `list`, `tuple` (неизменяемый), `dict`, `set` — соответствия `Array`/`Object`/`Map`/`Set`
- Срезы (`a[1:5:2]`), отрицательные индексы
- **List/dict/set comprehensions** — аналога в JS нет, освой как следует
- 🎯 *Докажи себе:* одним list comprehension собери словарь `{слово: длина}` из списка слов.

**0.1.3 Функции** ⚡
- Позиционные и keyword-аргументы, значения по умолчанию, `*args` / `**kwargs`
- `lambda`, функции как объекты первого класса
- Распаковка (`a, *rest = [...]`)
- 🎯 *Докажи себе:* напиши функцию с `*args, **kwargs`, которая логирует, чем её вызвали.

**0.1.4 ООП в Python** *(новое: не пропускай)*
- `class`, `__init__`, `self`; атрибуты класса vs экземпляра
- Dunder-методы (`__repr__`, `__eq__`, `__len__`, `__getitem__`)
- Наследование, `super()`, MRO (кратко)
- `@property`, `@staticmethod`, `@classmethod`
- `@dataclass` — главный рабочий инструмент для моделей данных
- 🎯 *Докажи себе:* сделай `@dataclass Point` с методом `__add__`, чтобы `p1 + p2` работало.

**0.1.5 Итераторы, генераторы, декораторы** *(новое: ключевые идиомы)*
- Итерируемые объекты и протокол итерации
- Генераторы и `yield`, ленивые вычисления
- Декораторы (что это и как писать свой)
- Контекстные менеджеры и `with`
- 🎯 *Докажи себе:* напиши генератор бесконечной последовательности Фибоначчи и возьми первые 10 через `itertools.islice`.

**0.1.6 Типизация и асинхронность** ⚡
- Type hints (`list[int]`, `Optional`, `dict[str, float]`), `typing`, `mypy`
- `async` / `await` — почти 1:1 с JS, но event loop устроен иначе
- 🎯 *Докажи себе:* добавь полную аннотацию типов к функции из 0.1.3 и прогони `mypy`.

**0.1.7 Модули, окружения, пакеты** ⚡
- `import`, пакеты, `__init__.py`, `if __name__ == "__main__"`
- `pip`, `venv`; `requirements.txt` / `pyproject.toml`; кратко про `uv`
- 🎯 *Докажи себе:* создай venv, поставь `requests`, зафиксируй в `requirements.txt`.

## Блок 0.2 — Научный стек (NumPy, Pandas, визуализация)

**Зачем:** NumPy — «как данные и веса живут в памяти», на нём стоит весь PyTorch; Pandas — инструмент работы с табличными данными. **Здесь не торопись.**

### Подтемы

**0.2.1 NumPy** *(основа основ)*
- `ndarray`, `dtype`, `shape`, `ndim`; чем массив отличается от списка
- Создание: `array`, `zeros`, `ones`, `arange`, `linspace`, `random`
- Индексация и срезы, булева индексация (маски), fancy indexing
- **Broadcasting** (правила выравнивания форм) — критично
- **Векторизация** vs циклы
- Оси (`axis=0/1`) и агрегации (`sum`, `mean`, `max`, `argmax`)
- `reshape`, `transpose`, конкатенация, `@` (матричное умножение)
- 🎯 *Докажи себе:* нормализуй матрицу 100×5 по столбцам **без единого цикла**.

**0.2.2 Pandas** *(ежедневный инструмент)*
- `Series` и `DataFrame`; чтение `read_csv` / `read_parquet`
- Индексация: `loc` (по меткам) vs `iloc` (по позициям)
- Фильтрация по условиям, добавление/удаление колонок
- Пропуски: `isna`, `fillna`, `dropna`
- `groupby` + агрегации; `merge` / `join` / `concat`
- `apply`, `map`, `value_counts`, `sort_values`
- Сводные таблицы (`pivot_table`)
- 🎯 *Докажи себе:* на любом CSV сделай `groupby` по категории и посчитай среднее по числовой колонке, отсортируй.

**0.2.3 Визуализация (Matplotlib + Seaborn)** ⚡ *(базы достаточно)*
- Matplotlib: `plot`, `scatter`, `hist`, `subplots`, подписи осей
- Seaborn: `histplot`, `boxplot`, `heatmap` корреляций, `pairplot`
- Когда какой график (распределение / связь / сравнение)
- 🎯 *Докажи себе:* построй гистограмму одной колонки и heatmap корреляций числовых колонок.

## Блок 0.4 — Твоё железо: что тянет Mac, что в облако

**Зачем:** сразу понимать, где запускать, чтобы не упереться и не переплатить. Справочный блок.

- **Mac (Apple Silicon, MPS-бэкенд PyTorch)** — годится для: всего Этапа 0, классического ML целиком (Этап 2), лёгких нейросетей и **инференса/дообучения небольших моделей**. MPS быстрее CPU, но медленнее и беднее по памяти, чем серьёзная GPU; часть операций может падать в CPU-фоллбэк.
- **Бесплатное облако (Colab / Kaggle)** — для: обучения сетей с нуля, экспериментов с трансформерами на учебных масштабах. Kaggle даёт ограниченные часы GPU/нед; Colab — бесплатный, но с лимитами.
- **Платная GPU понадобится** на: серьёзном дообучении LLM (даже LoRA на средних моделях), обучении трансформеров крупнее учебных, долгих прогонах. Варианты: Colab Pro, почасовая аренда (vast.ai, RunPod, облачные провайдеры). На критическом пути [A] это всплывёт только в опциональном fine-tuning (4.10) и при тяжёлом инференсе (5.1) — основную AI-инженерию (RAG, агенты) делаешь **без своей GPU**, через API.
- 🎯 *Докажи себе:* запусти `torch.backends.mps.is_available()` на Mace и сравни время умножения больших матриц на Mac MPS и на Colab GPU.
